关于BroadcastReceiver的几个错误

Menu

1.12-22 17:16:55.282 1481-1481/com.yuer_meterial.gentleni.musicplayer E/AndroidRuntime: java.lang.RuntimeException: Unable to instantiate receiver com.yuer_meterial.gentleni.musicplayer.MusicService$MusicReceiver: java.lang.IllegalAccessException: class com.yuer_meterial.gentleni.musicplayer.MusicService$MusicReceiver is not accessible from class android.app.ActivityThread
出现这个错误的原因是我的BroadcastReceiver是一个内部类，但是没有声明为public，访问权限出现错误，当我加上public后，又出现这个错误
12-22 17:19:12.967 4145-4145/com.yuer_meterial.gentleni.musicplayer E/AndroidRuntime: java.lang.RuntimeException: Unable to instantiate receiver com.yuer_meterial.gentleni.musicplayer.MusicService$MusicReceiver: java.lang.InstantiationException: class com.yuer_meterial.gentleni.musicplayer.MusicService$MusicReceiver has no zero argument constructor
说BroadcastReceiver没有零参的构造器，搜了一下，原因是 an (non-static) inner class cannot be instantiated by Android via the AndroidManifest.xml. You must dynamically register it, as shown in the link above.
就是说一个非静态的内部类是不能够通过AndroidManifest实例化的，你必须动态的注册，正好我是通过AndroidManifest注册的BroadcastReceiver，所以解决办法是通过代码动态的注册。

IntentFilter filter = new IntentFilter();
filter.addAction(MainActivity.CTL_ACTION);
registerReceiver(serviceReceiver, filter);

Android10_ContentProvider

Menu

####Content Provider

  • 应用程序之间共享数据的一种方式
  • 为存储和获取数据提供了统一的接口
  • 类似超市，向顾客提供各种商品，商品可能是来自各个厂商(例如数据库，sharedpreference等)
  • 数据可以存储于文件系统、SQLite等方式。内容提供者继承于ContentProvider基类，为其他应用程序取用和存储它管理的数据实现了一套标准方法。
  • 应用程序并不直接调用这些方法，而是使用ContentResolver对象，调用它的方法作为替代。

####UriMatcher

  • uri用于指定哪一个数据源，当一个数据源含有多个内容（比如多个表），就需要用不同的Uri进行区分

  • 这时候使用UriMatcher就可以帮助我们方便的过滤到TableA还是TableB, 然后进行下一步查询

  • UriMatcher本质上是一个文本过滤器，用在contentProvider中帮助我们过滤，分辨出查询者想要查询哪个数据表

What is Widget

窗口小部件，桌面上的n*n的小控件，常见的像音乐播放器控件，360加速等

如何创建

一个class三个xml

1.新建 TestWidget 继承自 AppWidgetProvider

可见 Widget是继承自BroadcastReceiver的。然后复写其onReceive方法和onUpdate方法，注意别忘了onUpdate方法，否则加载时候会提示加载小部件出现问题，加载不出来。

2.Widget的配置xml，setting_widget.xml

<appwidget-provider xmlns:android="http://schemas.android.com/apk/res/android"
                    android:layout_width="match_parent"
                    android:layout_height="match_parent"
                    android:initialLayout="@layout/layout_widget"
                    android:minHeight="140dp"
                    android:minWidth="140dp"
                    android:previewImage="@drawable/gege"
                    android:updatePeriodMillis="20000"
                    android:widgetCategory="home_screen"
    >

</appwidget-provider>

3.Widget的布局文件 layout_widget.xml

<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
  android:orientation="vertical"
  android:layout_width="match_parent"
  android:layout_height="match_parent">

<TextView
android:layout_width="wrap_content"
android:layout_height="wrap_content"
android:textAppearance="?android:attr/textAppearanceLarge"
android:text="Large Text"
android:id="@+id/widget_text_view"
android:layout_gravity="center_horizontal"/>

<Button
android:layout_width="wrap_content"
android:layout_height="wrap_content"
android:text="New Button"
android:id="@+id/widget_button"
android:layout_gravity="center_horizontal"/>
</LinearLayout>

4.在AndroidManifest.xml中注册

<receiver android:name=".TestWidget">
<intent-filter>
<action android:name="android.appwidget.action.APPWIDGET_UPDATE"/>
</intent-filter>
<meta-data android:name="android.appwidget.provider"
android:resource="@layout/setting_widget"
>
</meta-data>
</receiver>

Android06-BroadcastReceiver

Menu
  1. 注册

注册

  • 静态注册
  • 动态注册

1.SendBroadcastActivity.java

public class SendBroadcastActivity extends Activity {

public static final String COM_GENTLENI_TEST_BROADCAST = "com.gentleni.test.broadcast";
private TestBroadcastReceiver mTestBroadcastReceiver = new TestBroadcastReceiver();
private Button mButtonSendBroadcast;
@Override
protected void onCreate(Bundle savedInstanceState) {

    setContentView(R.layout.activity_sendbroadcast);
    mButtonSendBroadcast=(Button)findViewById(R.id.btn_send_broadcast);

    mButtonSendBroadcast.setOnClickListener(new View.OnClickListener() {
        @Override
        public void onClick(View v) {

            // 发送广播
            Intent intent = new Intent();
            intent.setAction(COM_GENTLENI_TEST_BROADCAST);
            intent.putExtra("toast", "this is my toast of broadcast");

            sendBroadcast(intent);
        }
    });
    super.onCreate(savedInstanceState);
}

@Override
protected void onStart() {
    super.onStart();
//        动态注册BroadcastReceiver
    IntentFilter intentFilter = new IntentFilter();
    intentFilter.addAction(COM_GENTLENI_TEST_BROADCAST);
    registerReceiver(mTestBroadcastReceiver, intentFilter);
}
}

2.BroadcastReceiver.java

public class TestBroadcastReceiver extends BroadcastReceiver{
    @Override
    public void onReceive(Context context, Intent intent) {

        if (intent != null) {
            if (TextUtils.equals(intent.getAction(), SendBroadcastActivity.COM_GENTLENI_TEST_BROADCAST)) {
                String toastString = intent.getStringExtra("toast");
                Toast.makeText(context, toastString, Toast.LENGTH_LONG).show();
            }
        }
    }
}

3.AndroidManifest.xml

<receiver android:name=".TestBroadcastReceiver">
           <intent-filter>
               <action android:name="com.gentleni.test.broadcast"></action>
           </intent-filter>
       </receiver>
